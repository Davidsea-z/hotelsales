# CSV导出乱码问题修复说明

## 🐛 问题描述

**症状**：导出的CSV文件用Excel打开显示乱码

**原因**：
1. Excel默认使用GBK/GB2312编码打开CSV文件
2. 我们导出的CSV文件使用UTF-8编码
3. 编码不匹配导致中文显示为乱码

---

## ✅ 解决方案

### 修复方法：添加UTF-8 BOM

**什么是BOM？**
- BOM = Byte Order Mark（字节顺序标记）
- UTF-8 BOM = `\uFEFF`（三个字节：EF BB BF）
- Excel通过BOM识别文件编码

**修复代码**：
```javascript
// 添加UTF-8 BOM
const BOM = '\uFEFF';
const csvWithBOM = BOM + csv;

// 创建Blob时指定UTF-8编码
const blob = new Blob([csvWithBOM], { type: 'text/csv;charset=utf-8;' });
```

### 其他改进

**1. 正确处理逗号和引号**
```javascript
// 用双引号包裹字段，避免逗号分隔问题
`"${c.name}"`

// 转义双引号（CSV标准：两个双引号表示一个）
`"${(c.message || '').replace(/"/g, '""')}"`
```

**2. 状态中文显示**
```javascript
const statusMap = {
    'new': '待跟进',
    'contacted': '已联系',
    'converted': '已转化',
    'lost': '已流失'
};
```

**3. Windows换行符**
```javascript
// 使用\r\n（CRLF）而不是\n（LF）
.join('\r\n')
```

## 🔧 技术细节

### UTF-8 BOM的作用
```
文件开头字节：EF BB BF（UTF-8 BOM）
              ↓
Excel识别：哦，这是UTF-8编码的文件
              ↓
结果：中文正确显示
```

### CSV格式规范（RFC 4180）
1. **字段包含逗号** → 用双引号包裹
2. **字段包含引号** → 用两个双引号转义
3. **字段包含换行** → 用双引号包裹
4. **Windows系统** → 使用CRLF换行（\r\n）

## 🎓 知识扩展

### 为什么Excel这么特殊？
- Excel优先使用系统默认编码（中文Windows = GBK）
- 只有看到UTF-8 BOM才会使用UTF-8编码
- 其他软件（如Google Sheets）默认使用UTF-8

### UTF-8 vs UTF-8 with BOM
- **UTF-8**：无BOM，更通用，适合Web
- **UTF-8 with BOM**：有BOM，Excel友好，适合导出
